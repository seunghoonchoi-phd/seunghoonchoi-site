# 리드패스트(ReadFast): 읽기 속도와 이해 훈련

같은 글을 두 번째 읽으면 대개 더 빨라집니다. 하지만 처음 보는 글에서도 내용을 놓치지 않아야 읽기 실력이 좋아졌다고 말할 수 있습니다.

리드패스트는 영어와 중국어 읽기를 연습하는 설치형 웹앱(PWA)입니다. 읽은 속도와 이해 결과를 따로 기록하고, 연습에 쓰지 않은 글에서도 결과가 이어지는지 확인합니다.

## 앱이 기록하는 것

- 속도: 영어는 분당 단어 수(WPM), 중국어는 분당 글자 수(CPM)로 기록합니다.
- 이해: 맞힌 문항 수와 문항 유형별 결과를 속도 옆에 따로 보여줍니다.
- 조건: 처음 보는 글인지, 도움을 받았는지, 끝까지 마쳤는지 함께 저장합니다.
- 피로: 사용자가 느낀 피로를 기록해 다음 난이도를 정할 때 참고합니다.
- 목적: 정확히 읽기, 핵심 잡기, 정보 찾기를 서로 다른 과제로 다룹니다.

속도와 이해를 곱한 ERR 점수는 대표 지표로 쓰지 않습니다. 서로 다른 두 결과를 한 숫자로 합치면 빨라진 이유와 이해가 떨어진 이유를 구분하기 어렵기 때문입니다.

## 훈련은 네 단계로 돕니다

1. 현재 상태 확인: 처음 보는 글을 도움 없이 읽습니다.
2. 약한 부분 연습: 어휘, 문장 처리, 반복읽기 가운데 지금 필요한 한 가지를 고릅니다.
3. 새 글에서 확인: 연습에 쓰지 않은 글을 읽어 익숙함과 실제 변화를 구분합니다.
4. 주간 재측정: 최근 기록을 모아 다음 주 난이도와 연습 항목을 정합니다.

오늘 화면에는 약 10분 분량의 세 블록만 보여줍니다. 짧은 준비와 복습, 주 훈련, 처음 보는 글 확인 순서입니다. 훈련 탭에서는 모든 드릴을 언제든 직접 열 수 있으며, 앞 단계를 끝내야 다음 드릴이 열리는 잠금은 없습니다.

## 난이도와 다음 속도를 정하는 방법

앱 난이도는 1부터 6까지입니다. 이 숫자는 앱 안에서 글을 고르기 위한 훈련 구간입니다. CEFR이나 HSK 점수를 인증하거나 대신하지 않습니다. 시작 난이도도 처음 보는 글을 여러 번 읽은 뒤 임시로 추천합니다.

다음 속도 조정에는 아래 초기 제품 규칙을 씁니다.

| 최근의 처음 보는 글 결과 | 다음 조정 |
|---|---|
| 이해 80% 이상 2회, 피로 낮음 | 목표 속도를 약 5% 올림 |
| 이해 60% 이상 80% 미만 | 현재 목표 유지 |
| 이해 60% 미만 또는 높은 피로 | 목표 속도만 약 5% 낮춤 |

이 구간과 5% 값은 연구가 정한 자연법칙이 아닙니다. 앱이 한 번에 너무 크게 움직이지 않도록 만든 초기 운영 규칙입니다. 사용자 기록이 쌓이면 실제 결과에 맞춰 고칠 수 있습니다.

대표 속도는 최근 기록 가운데 다음 조건을 만족한 시도의 중앙값입니다.

- 연습에 쓰지 않은 글
- 도움 없이 읽은 시도
- 끝까지 마친 시도
- 이해 80% 이상

조건을 만족한 기록이 적으면 앱은 대표 속도를 확정하지 않고 더 측정하라고 안내합니다.

## 드릴의 역할

핵심 훈련:

- 제한 시간을 둔 읽기와 이해 확인
- 같은 글 반복읽기 뒤 새 글 확인
- 어휘 간격 복습
- 정확히 읽기, 핵심 잡기, 정보 찾기

언어별 보조 훈련:

- 영어 구절 안내
- 중국어 글자 인지와 단어 경계 연습

연습 도구:

- 문장 확인, 문맥 추론, 미리보기
- 덮고 떠올리기와 자기설명
- 논문을 세 번에 나눠 읽는 방법

보조 훈련과 도구는 연결된 글 읽기를 대신하지 않습니다. 필요한 사람과 목적에 맞춰 골라 씁니다.

## 연구에서 가져온 것과 제품이 정한 것

연구가 뒷받침하는 범위:

- 읽는 속도를 무리하게 올리면 이해가 떨어질 수 있습니다.
- 되돌아 읽기와 읽는 사람의 시간 조절은 어려운 글을 이해하는 데 도움이 될 수 있습니다.
- 같은 글을 반복하면 그 글의 읽기는 좋아질 수 있습니다. 다른 글로 이어지는 정도는 연구 대상과 조건에 따라 다릅니다.
- 간격을 둔 복습과 인출 연습은 학습 내용을 오래 기억하는 데 도움이 됩니다.

앱이 운영을 위해 정한 범위:

- 이해 80%를 대표 속도에 포함하는 기준으로 씁니다.
- 최근 시도 두 번과 피로도를 보고 다음 목표를 조절합니다.
- 한 번에 속도와 난이도를 함께 올리거나 내리지 않습니다.
- 앱 난이도 1부터 6을 지문 선택에 씁니다.

자세한 설명과 출처는 앱의 원리 화면에서 확인할 수 있습니다.

## 개인정보와 저장

- 계정과 서버 동기화가 없습니다.
- 훈련 기록과 붙여넣은 글은 현재 브라우저의 로컬 저장소에 보관합니다.
- 앱 안에는 제3자 분석 스크립트가 없습니다.
- 붙여넣은 글의 본문을 네트워크로 보내지 않습니다.
- 내보내기를 누르면 사용자가 직접 백업 파일을 만듭니다.

브라우저 데이터를 지우거나 저장 공간이 부족하면 로컬 기록이 사라질 수 있습니다. 중요한 기록은 설정에서 정기적으로 내보내는 편이 안전합니다. 저장에 실패하면 앱이 화면에 오류를 알려야 하며, 성공한 것처럼 조용히 넘어가면 안 됩니다.

## 접근성과 읽기 화면

- 키보드로 주요 화면과 동작을 이용할 수 있습니다.
- 주 동작 영역은 최소 44px이며, 키보드 포커스를 눈으로 확인할 수 있습니다.
- 밝은 화면과 어두운 화면을 지원합니다.
- 본문 글자 크기, 줄 간격, 읽기 폭, 움직임 줄이기, 타이머 설정을 조절할 수 있습니다.
- 한국어와 영어 화면을 지원합니다. 화면 언어와 훈련할 언어(영어 또는 중국어)는 별도 설정입니다.
- 세로 화면을 강제하지 않으며, 모바일과 확대 화면에서 내용이 다시 배치됩니다.

## 오프라인 사용

온라인에서 앱을 한 번 정상적으로 연 뒤에는 앱 셸, 드릴 코드, 지문과 어휘 데이터를 오프라인에서도 열 수 있습니다. 서비스워커는 이 앱 경로의 같은 출처 정적 GET 요청만 캐시합니다. 다른 사이트 요청과 붙여넣은 글은 캐시하지 않습니다. 런타임 캐시는 항목 수를 제한합니다.

## 프로젝트 구조

```text
index.html              앱 셸과 접근성 표지
css/styles.css          색, 반응형 배치, 읽기 설정, 포커스 상태
js/app.js               라우터와 화면 구성
js/i18n.js              한국어와 영어 UI 기반
js/store.js             로컬 상태, 시도 기록, 가져오기와 내보내기
js/content.js           지문 선택과 처음 보는 글 관리
js/metrics.js           속도와 이해를 분리한 지표
js/program.js           네 단계 훈련 순서와 다음 추천
js/levels.js            앱 난이도 1부터 6
js/progression.js       네 단계 진행 상태
js/theory.js            원리와 근거 자료
js/drills/              핵심 훈련, 언어별 보조, 연습 도구
data/                   영어와 중국어 지문, 어휘, 분할 데이터
manifest.webmanifest    설치 정보
sw.js                   오프라인 정적 파일 캐시
_build/                 로컬 서버, 데이터 도구, 자동 검사
```

기존 `readfast.v2` 데이터는 지우지 않습니다. 새 시도 기록으로 옮길 수 있는 값만 보존하고, 과거의 ERR나 정복 도장 기록을 새 숙달 판정에 쓰지 않습니다.

## 로컬 실행과 검사

ES 모듈과 데이터 요청을 쓰므로 `file://`로 열지 말고 로컬 HTTP 서버를 사용합니다.

```powershell
py _build/serve.py 5577
```

브라우저에서 `http://127.0.0.1:5577`을 엽니다.

정적 파일과 오프라인 목록 검사:

```powershell
node _build/test-assets.mjs
```

핵심 로직과 드릴 테스트 파일이 있으면 함께 실행합니다.

```powershell
node _build/test-core.mjs
node _build/test-drills.mjs
```

GitHub Pages 배포 작업은 JavaScript 구문, 로직 테스트, 오프라인 파일 목록, 앱 안의 제3자 스크립트 금지를 먼저 확인합니다. 검사가 끝난 뒤 기존 Pages 배포 절차를 실행합니다.
