AI와 대화를 많이 할수록, 역설적으로 손에 남는 지식은 줄어든다. 이유는 단순하다. AI의 생성량이 내가 검증하고 정리할 수 있는 양을 넘어서기 때문이다. 좋은 말은 쏟아지는데, 한 달 뒤 다시 찾을 수 있는 건 거의 없다.

그래서 "AI로 노트 정리하기" 시도는 대부분 실패한다. 다들 *더 많이 뽑는* 데 집중하기 때문이다. 많이 뽑으면 노트는 금세 쓰레기통이 된다. 다시 안 보는 메모가 9할이다.

나는 반대로 갔다. 많이 뽑는 도구가 아니라, 독하게 버리는 도구를 만들었다. 이 시스템의 진짜 핵심은 추출 능력이 아니라 *버리는 기준*이다.

## 추출이 아니라 폐기가 먼저다

내 추출 규율의 1순위는 전부 "버려라"로 시작한다.

- 애매하면 버린다
- 과잉 추출 금지, 대화 요약 금지, 일반론 금지
- 검색으로 쉽게 복구되는 정보는 버린다 (이게 가장 강력한 필터다)
- 고유명사를 빼면 흔한 일반론이 되는 내용은 버린다
- 저장할 게 없으면 "저장할 내용 없음"만 출력한다
- **과잉 추출보다 누락이 낫다**

남기는 건 딱 여섯 가지에 명확히 해당할 때뿐이다. 실제로 해야 할 행동, 처음 하는 사람은 모를 정보, 시스템의 구조와 운영 방식, 그대로 재사용할 수 있는 문구, 증빙이 될 사실, 그리고 판단 기준을 바꿀 정도의 깨달음.

## 왜 한 번에 안 하고 3단으로 나누나

LLM에게 "대화에서 중요한 거 정리해줘"라고 한 번에 시키면, 모델은 친절하게도 다 정리해준다. 그게 문제다. 추출·해석·정리를 한 번에 시키면 폐기가 작동하지 않는다.

그래서 단계를 쪼개고, 각 단계마다 다시 버리게 만든다. 1차는 후보만 추출한다(정리하지 않는다). 2차는 후보를 해석해 의미를 부여하되, 추론이 들어가면 "추정"이라고 표시하게 한다. 3차에서 노션에 바로 붙여넣을 계층형 bullet로 정리하며 마지막으로 한 번 더 버린다. 살아남은 항목은 맥락 없이 단독으로 읽어도 이해되어야 한다.

핵심 장치는 "추정 여부를 표시"하게 만든 것이다. AI가 지어낸 구조와 대화에 실제 근거가 있는 구조를 섞지 않게 강제한다.

## 복붙용 프롬프트

### 1차 — 후보 추출

```text
아래 대화/텍스트에서 저장할 "후보"만 추출해라.
최종 정리가 아니라 1차 후보 추출이다. 분류를 다듬거나 문장을 꾸미지 마라.
핵심은 저장 가치가 낮은 것을 강하게 버리고, 2차 해석할 가치가 있는 후보만 남기는 것이다.

[최우선 원칙]
- 애매하면 버려라 / 과잉 추출 금지 / 대화 요약 금지 / 일반론 금지
- 검색으로 쉽게 복구 가능한 정보는 버려라
- 없는 내용 지어내지 말라
- 저장 가치가 없으면 "저장할 내용 없음"만 출력하라
- 항목 수는 적게, 같은 뜻의 반복은 합쳐라

[남길 것] 아래 중 하나에 명확히 해당할 때만
1) 실제로 해야 할 행동  2) 처음 하는 사람은 모를 중요한 정보
3) 시스템의 구조·흐름·분기·운영 방식  4) 그대로 재사용 가능한 문구
5) 증빙으로 쓸 사실  6) 판단 기준을 바꿀 깨달음

[버릴 것] 감상 / 맞장구 / 중간 설명 / 단순 답변 요약 / 보편 지식 /
구조를 드러내지 않는 단편 / 고유명사를 빼면 일반론이 되는 내용

[출력] 분류하지 말고 후보만 나열. 각 후보에 라벨 1개만 임시 부여:
[행동] [숨은정보] [구조] [문구] [증빙] [깨달음]
- 각 항목 1~3문장 / bullet만 / 문구 후보는 원문 그대로 보존
```

### 2차 — 의미 부여

```text
위는 1차 후보 추출 결과다. 이제 각 후보를 해석해 "의미"를 부여해라.
아직 예쁘게 적재하는 단계가 아니다. 무엇이고, 왜 중요하고, 어떤 구조를 드러내는지 해석하라.

[원칙]
- 1차 후보 중에서도 가치가 낮으면 여기서 다시 버려라
- 구조·운영원리는 대화에 직접 근거가 있는 범위에서만 추론하라
- 추론이 개입되면 "추정"이라고 표시하라
- 일반론 수준이면 버려라

각 후보를 아래 최종 유형 1개로 판정하라:
Action ITEM / 대상의 본질·원리 / 핵심정보 / 문구 / 증빙 / 깨달음 / 기타

[출력 형식] 후보 제목
  - 최종유형: …
  - 핵심 의미: …
  - 왜 중요한지: …
  - 구조/운영원리: … (해당 시)
  - 추정 여부: 직접 근거 / 일부 추정
```

### 3차 — 노션 적재용 최종 정리

```text
위는 2차 의미 부여 결과다. 노션에 바로 옮길 수 있게 최종 정리하라.
저장 가치가 낮은 항목은 마지막으로 한 번 더 버려라.

[원칙] 애매하면 버려라 / 과잉 추출 금지 / 일반론 금지 /
분류를 채우려고 억지로 항목을 남기지 말라 / 없으면 "저장할 내용 없음"

[분류] Action ITEM / 대상의 본질·원리 / 핵심정보 / 문구 / 깨달음 / 기타
- 저장 가치 있는 항목만 남기고, 각 항목에 분류 1개만 부여
- 항목 없는 분류는 생략, 같은 분류끼리 묶어 출력

[출력] 모든 항목은 계층형 bullet
- 상위 bullet은 제목만, 설명은 하위 bullet
- 살아남은 항목은 단독으로 읽어도 이해되게, 쉬운 말로
- [대상의 본질·원리]는 "본질: X는 ~하는 기관/시스템/구조다"로 시작
- [문구]는 제목 + 한글 뜻 + 원문 (중국어는 병음 추가)
```

중요한 대화는 셋을 나눠 돌리는 걸 권한다. 한 번에 시키면 폐기 강도가 떨어진다.

## 끝으로

이 시스템의 가치는 화려한 출력이 아니라, "무엇을 안 남길지"를 기계가 대신 판단하게 만든 규율에 있다. AI는 점점 더 많이 만들어낼 것이고, 그럴수록 희소해지는 건 생성이 아니라 선별과 폐기다. 나는 그 폐기 기준을 코드처럼 버전 관리한다.
