AI랑 대화를 많이 할수록, 이상하게 손에 남는 지식은 줄어든다.

이유는 간단하다. AI가 쏟아내는 양이, 내가 확인하고 정리할 수 있는 양을 넘어선다. 좋은 말은 계속 나온다. 그런데 한 달 뒤에 다시 찾을 수 있는 건 거의 없다.

그래서 "AI로 노트 정리하기"는 대부분 망한다. 다들 더 많이 뽑는 데만 매달린다. 많이 뽑을수록 노트는 금방 쓰레기통이 된다. 다시 안 보는 메모가 열에 아홉이다.

나는 반대로 갔다. 많이 뽑는 도구가 아니라, 독하게 버리는 도구를 만들었다. 이 시스템의 진짜 핵심은 추출 능력이 아니라 버리는 기준이다.

## 뽑기 전에 버려라. 검색하면 나오는 건 다 지운다

내 추출 규칙은 처음부터 끝까지 전부 "버려라"로 시작한다.

- 애매하면 버린다
- 과하게 뽑지 마라, 대화 요약하지 마라, 일반론 적지 마라
- 검색하면 금방 나오는 정보는 버린다 (이게 제일 센 필터다)
- 고유명사 빼면 그냥 흔한 소리가 되는 건 버린다
- 저장할 게 없으면 "저장할 내용 없음"만 출력한다
- **과하게 뽑느니 빠뜨리는 게 낫다**

남기는 건 딱 여섯 가지에 또렷이 들어맞을 때뿐이다. 실제로 해야 할 행동, 처음 하는 사람은 모르는 정보, 시스템이 어떻게 굴러가는지 보여주는 구조, 그대로 다시 쓸 수 있는 문구, 증거가 되는 사실, 그리고 판단 기준을 바꿀 만한 깨달음.

## 한 번에 시키면 AI가 다 정리해버린다. 그래서 3번 쪼개 매번 또 버린다

LLM한테 "대화에서 중요한 거 정리해줘" 한 번에 시켜봐라. 모델은 친절하게 다 정리해준다. 그게 문제다. 뽑기랑 해석이랑 정리를 한꺼번에 시키면, 버리는 일이 작동을 안 한다.

그래서 단계를 쪼갠다. 단계마다 또 버리게 만든다.

1차는 후보만 뽑는다. 정리는 안 한다. 2차는 후보에 의미를 붙이되, 추측이 끼면 "추정"이라고 표시하게 한다. 3차는 노션에 바로 붙여넣을 계층형 bullet로 정리하면서, 마지막으로 한 번 더 버린다. 살아남은 항목은 앞뒤 맥락 없이 혼자 읽어도 알아먹게 만든다.

핵심 장치는 "추정 여부 표시"다. AI가 지어낸 구조와 대화에 진짜 근거가 있는 구조가 섞이지 않게 강제로 갈라놓는다.

## 복붙용 프롬프트. 그대로 긁어 쓰면 된다

### 1차: 정리 말고 후보만 뽑기

```text
아래 대화/텍스트에서 저장할 "후보"만 추출해라.
최종 정리가 아니라 1차 후보 추출이다. 분류를 다듬거나 문장을 꾸미지 마라.
핵심은 저장 가치가 낮은 것을 강하게 버리고, 2차 해석할 가치가 있는 후보만 남기는 것이다.

[최우선 원칙]
- 애매하면 버려라 / 과잉 추출 금지 / 대화 요약 금지 / 일반론 금지
- 검색으로 쉽게 복구 가능한 정보는 버려라
- 없는 내용 지어내지 말라
- 저장 가치가 없으면 "저장할 내용 없음"만 출력하라
- 항목 수는 적게, 같은 뜻의 반복은 합쳐라

[남길 것] 아래 중 하나에 명확히 해당할 때만
1) 실제로 해야 할 행동  2) 처음 하는 사람은 모를 중요한 정보
3) 시스템의 구조·흐름·분기·운영 방식  4) 그대로 재사용 가능한 문구
5) 증빙으로 쓸 사실  6) 판단 기준을 바꿀 깨달음

[버릴 것] 감상 / 맞장구 / 중간 설명 / 단순 답변 요약 / 보편 지식 /
구조를 드러내지 않는 단편 / 고유명사를 빼면 일반론이 되는 내용

[출력] 분류하지 말고 후보만 나열. 각 후보에 라벨 1개만 임시 부여:
[행동] [숨은정보] [구조] [문구] [증빙] [깨달음]
- 각 항목 1~3문장 / bullet만 / 문구 후보는 원문 그대로 보존
```

### 2차: 뜻 붙이고, AI가 지어낸 건 '추정' 도장 찍기

```text
위는 1차 후보 추출 결과다. 이제 각 후보를 해석해 "의미"를 부여해라.
아직 예쁘게 적재하는 단계가 아니다. 무엇이고, 왜 중요하고, 어떤 구조를 드러내는지 해석하라.

[원칙]
- 1차 후보 중에서도 가치가 낮으면 여기서 다시 버려라
- 구조·운영원리는 대화에 직접 근거가 있는 범위에서만 추론하라
- 추론이 개입되면 "추정"이라고 표시하라
- 일반론 수준이면 버려라

각 후보를 아래 최종 유형 1개로 판정하라:
Action ITEM / 대상의 본질·원리 / 핵심정보 / 문구 / 증빙 / 깨달음 / 기타

[출력 형식] 후보 제목
  - 최종유형: …
  - 핵심 의미: …
  - 왜 중요한지: …
  - 구조/운영원리: … (해당 시)
  - 추정 여부: 직접 근거 / 일부 추정
```

### 3차: 노션에 바로 붙일 모양으로 만들고 마지막에 한 번 더 버리기

```text
위는 2차 의미 부여 결과다. 노션에 바로 옮길 수 있게 최종 정리하라.
저장 가치가 낮은 항목은 마지막으로 한 번 더 버려라.

[원칙] 애매하면 버려라 / 과잉 추출 금지 / 일반론 금지 /
분류를 채우려고 억지로 항목을 남기지 말라 / 없으면 "저장할 내용 없음"

[분류] Action ITEM / 대상의 본질·원리 / 핵심정보 / 문구 / 깨달음 / 기타
- 저장 가치 있는 항목만 남기고, 각 항목에 분류 1개만 부여
- 항목 없는 분류는 생략, 같은 분류끼리 묶어 출력

[출력] 모든 항목은 계층형 bullet
- 상위 bullet은 제목만, 설명은 하위 bullet
- 살아남은 항목은 단독으로 읽어도 이해되게, 쉬운 말로
- [대상의 본질·원리]는 "본질: X는 ~하는 기관/시스템/구조다"로 시작
- [문구]는 제목 + 한글 뜻 + 원문 (중국어는 병음 추가)
```

중요한 대화는 셋으로 나눠 돌려라. 한 번에 시키면 버리는 강도가 떨어진다.

## 값나가는 건 화려한 출력이 아니라 '안 남길 걸 고르는 규칙'이다

이 시스템의 값어치는 화려한 출력에 있지 않다. "무엇을 안 남길지"를 기계가 대신 판단하게 만든 규칙에 있다.

AI는 앞으로 점점 더 많이 만들어낼 거다. 그럴수록 귀해지는 건 생성이 아니라, 골라내고 버리는 일이다. 그래서 나는 그 버리는 기준을 코드처럼 버전으로 관리한다.
