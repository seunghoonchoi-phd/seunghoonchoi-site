AIとたくさん話すほど、不思議と手元に残る知識は減っていきます。

理由は単純です。AIが吐き出す量が、自分で確認して整理できる量を超えてしまうから。いい話はどんどん出てきます。でも一か月後にもう一度たどり着けるものは、ほとんどありません。

だから「AIでノートを整理する」は、たいてい失敗します。みんな、もっとたくさん抽出することばかりに気を取られます。たくさん抽出するほど、ノートはあっという間に使えないメモであふれます。二度と見返さないメモが十中八九です。

私は逆をいきました。たくさん抽出する道具ではなく、容赦なく捨てる道具を作ったのです。このシステムの本当の肝は、抽出する力ではなく、捨てる基準にあります。

## 検索すれば出てくるものは全部捨てろ

私の抽出ルールは、最初から最後まですべて「捨てろ」で始まります。

- あいまいなら捨てる
- 抽出しすぎるな、会話を要約するな、一般論を書くな
- 検索すればすぐ出てくる情報は捨てる（これが一番強いフィルターです）
- 固有名詞を抜くとただのありふれた話になるものは捨てる
- 保存するものがなければ「保存する内容なし」とだけ出力する
- 抽出しすぎるくらいなら、取りこぼすほうがいい

残すのは、次の六つにはっきり当てはまるときだけ。実際にやるべき行動、初めての人には分からない情報、システムがどう回っているかを見せる構造、そのまま使い回せる文言、証拠になる事実、そして判断基準を変えるほどの気づき。

## 一度にまとめて任せると、AIは全部生かしてしまう

LLMに「会話から大事なところを整理して」と一度に任せてみてください。モデルは親切に全部整理してくれます。それが問題なのです。抽出と解釈と整理を一度に任せると、捨てるという働きが回りません。

だから段階を分けます。段階ごとに、また捨てさせます。

一段目は候補だけを抜き出します。整理はしません。二段目は候補に意味を付けますが、推測が混じったら「推定」と表示させます。三段目はNotionにそのまま貼り付けられる階層式の箇条書きに整理しながら、最後にもう一度捨てます。生き残った項目は、前後の文脈なしで単独で読んでも分かるようにします。

肝になる仕掛けは「推定かどうかの表示」です。AIが作り上げた構造と、会話に本当に根拠がある構造とが混ざらないように、強制的に切り分けます。

## コピペ用プロンプト、そのまま使ってください

### 一段目：整理せず候補だけ抜く

```text
以下の会話・テキストから、保存すべき「候補」だけを抽出せよ。
最終的な整理ではなく、一次の候補抽出だ。分類を整えたり文章を飾ったりするな。
肝は、保存価値の低いものを強く捨て、二次で解釈する価値のある候補だけを残すことだ。

[最優先の原則]
- あいまいなら捨てろ / 抽出しすぎ禁止 / 会話の要約禁止 / 一般論禁止
- 検索で簡単に復元できる情報は捨てろ
- 存在しない内容を作り上げるな
- 保存価値がなければ「保存する内容なし」とだけ出力せよ
- 項目数は少なく、同じ意味の繰り返しはまとめよ

[残すもの] 以下のいずれかに明確に当てはまるときだけ
1) 実際にやるべき行動  2) 初めての人には分からない重要な情報
3) システムの構造・流れ・分岐・運用のしかた  4) そのまま使い回せる文言
5) 証拠として使える事実  6) 判断基準を変える気づき

[捨てるもの] 感想 / 相づち / 途中の説明 / 単なる回答の要約 / 一般常識 /
構造を示さない断片 / 固有名詞を抜くと一般論になる内容

[出力] 分類せず、候補だけを列挙。各候補にラベルを1つだけ仮付け:
[行動] [隠れ情報] [構造] [文言] [証拠] [気づき]
- 各項目は1〜3文 / 箇条書きのみ / 文言の候補は原文のまま保存
```

### 二段目：意味を付け、作り話には推定を打つ

```text
上は一次の候補抽出結果だ。これから各候補を解釈し、「意味」を与えよ。
まだきれいに積み上げる段階ではない。何であり、なぜ重要で、どんな構造を示すのかを解釈せよ。

[原則]
- 一次候補の中でも価値が低ければ、ここで再び捨てろ
- 構造・運用原理は、会話に直接の根拠がある範囲だけで推論せよ
- 推論が入ったら「推定」と表示せよ
- 一般論のレベルなら捨てろ

各候補を、以下の最終タイプの1つに判定せよ:
Action ITEM / 対象の本質・原理 / 中核情報 / 文言 / 証拠 / 気づき / その他

[出力形式] 候補のタイトル
  - 最終タイプ: …
  - 中核の意味: …
  - なぜ重要か: …
  - 構造/運用原理: …（該当する場合）
  - 推定かどうか: 直接の根拠 / 一部推定
```

### 三段目：Notionの形に整え、また捨てる

```text
上は二次の意味付け結果だ。Notionにそのまま移せるよう、最終整理せよ。
保存価値の低い項目は、最後にもう一度捨てろ。

[原則] あいまいなら捨てろ / 抽出しすぎ禁止 / 一般論禁止 /
分類を埋めるために無理に項目を残すな / なければ「保存する内容なし」

[分類] Action ITEM / 対象の本質・原理 / 中核情報 / 文言 / 気づき / その他
- 保存価値のある項目だけを残し、各項目に分類を1つだけ付与
- 項目のない分類は省略、同じ分類どうしをまとめて出力

[出力] すべての項目は階層式の箇条書き
- 上位の箇条書きはタイトルのみ、説明は下位の箇条書き
- 生き残った項目は、単独で読んでも分かるように、やさしい言葉で
- [対象の本質・原理]は「本質: Xとは〜する器官/システム/構造だ」で始める
- [文言]はタイトル + 日本語の意味 + 原文（中国語は併音を追加）
```

大事な会話は、三つに分けて回してください。一度にまとめて任せると、捨てる強さが落ちます。

## 値打ちがあるのは、捨てるルールのほうだ

このシステムの値打ちは、派手な出力にあるのではありません。「何を残さないか」を機械に代わりに判断させたルールにあります。

AIはこれからどんどん多くを生み出していきます。そうなるほど貴重になるのは、生成ではなく、選り分けて捨てる仕事です。だから私は、その捨てる基準を、版を分けてバージョン管理しています。
