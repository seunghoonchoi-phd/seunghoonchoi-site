<p class="inline-image-caption">即使您收集了大量信息，如果没有丢弃这些信息的标准，那么将其组织起来也只是另一种噪音。</p>

跟 AI 聊得越多，奇怪的是，手里真正留下的知识反而越少。

道理很简单。AI 一下子吐出来的量，超过了我能核对、能整理的量。有用的话不停地出现。可一个月以后还能再找回来的，几乎没有。

所以大多数人"用 AI 整理笔记"，最后都黄了。大家全都死磕在怎么多抽一点上。抽得越多，笔记越快堆满没用的东西。十条备忘里有九条，再也不会被翻开。

我反着来。我做的不是抽得更多的工具，而是一个狠下心丢的工具。这套系统真正的核心，不是提取能力，而是丢弃的标准。

## 搜一下就有的，全扔

我这套提取规则，从头到尾都是从"扔掉"开始的。

- 拿不准就扔
- 别过度提取，别给对话写摘要，别记泛泛而谈
- 搜一下马上就能找到的信息，扔掉（这是最狠的一道过滤）
- 去掉专有名词就变成大白话套话的，扔掉
- 没东西可存，就只输出一句"无内容可保存"
- 宁可漏掉，也别过度提取

留下来的只有六样，而且只在它清清楚楚对得上时才留：真正要去做的行动、新手不会知道的信息、能看出某套系统怎么运转的结构、能原样照搬再用的表述、可以当证据的事实，还有足以改变你判断标准的领悟。

## 一口气让它做完，AI 啥都给你留

你试试让 LLM"把这段对话里重要的整理一下"，一口气交代下去。模型会很热心地把一切都给你整理出来。坏就坏在这儿。提取、解读、整理一口气全做完，丢弃这一步根本就不会启动。

所以我把它拆成几段，每一段都逼它再扔一次。

第一段只抽候选，不整理。第二段给候选加上含义，但凡掺了猜的，就得标成"推测"。第三段整理成可以直接粘进 Notion 的层级 bullet，顺手再扔最后一次。留下来的条目，要做到脱离前后文、单独读也能看懂。

最关键的那一步，就是"标出是不是推测"。它强制把 AI 自己编出来的结构，和对话里真有依据的结构，明确分开，不让它们混在一起。

## 复制即用的提示词，直接抄走

### 第一段：别整理，只抽候选

```text
从下面的对话/文本里，只抽出值得保存的"候选"。
这是第一遍候选提取，不是最终成稿。别去打磨分类，也别润色句子。
关键是狠狠丢掉低价值的东西，只留下值得在第二段去解读的候选。

[最高原则]
- 拿不准就扔 / 不过度提取 / 不做对话摘要 / 不留泛泛而谈
- 搜一下就能轻松找回的信息，扔掉
- 没有的内容不要编造
- 没有值得保存的，就只输出"无内容可保存"
- 条目数量要少，意思相同的合并掉

[要留的] 仅当它明确符合下列其一时
1) 真正要去做的行动  2) 新手不会知道的重要信息
3) 某套系统的结构 / 流程 / 分支 / 运作方式  4) 能原样复用的表述
5) 可以当证据的事实  6) 改变判断标准的领悟

[要丢的] 感想 / 附和 / 中间过渡的解释 / 单纯的答复摘要 / 通用常识 /
不显露结构的零碎片段 / 去掉专有名词就变成套话的内容

[输出] 不要分类，只把候选列出来。每条临时贴一个标签：
[行动] [隐藏信息] [结构] [表述] [证据] [领悟]
- 每条 1~3 句 / 只用 bullet / 表述类候选原文照搬，一字不改
```

### 第二段：加上含义，编出来的标"推测"

```text
上面是第一段的候选提取结果。现在逐条解读每个候选，给它赋予"含义"。
这还不是漂漂亮亮码放归档的阶段。要解读它是什么、为什么重要、显露出什么结构。

[原则]
- 哪怕是第一段的候选，价值低的在这一步再丢一次
- 结构、运作原理只在对话直接支撑的范围内推断
- 只要掺进了推断，就标成"推测"
- 落到泛泛而谈这个层次的，扔掉

把每个候选判定为下列最终类型中的唯一一种：
Action item / 对象的本质与原理 / 核心信息 / 表述 / 证据 / 领悟 / 其他

[输出格式] 候选标题
  - 最终类型: ……
  - 核心含义: ……
  - 为什么重要: ……
  - 结构 / 运作原理: ……（适用时）
  - 是否推测: 直接有据 / 部分推测
```

### 第三段：做成 Notion 的样子，再扔一遍

```text
上面是第二段赋予含义的结果。现在做成可以直接挪进 Notion 的最终整理。
低价值的条目，最后再丢一次。

[原则] 拿不准就扔 / 不过度提取 / 不留泛泛而谈 /
不要为了填满分类硬留条目 / 没有就输出"无内容可保存"

[分类] Action item / 对象的本质与原理 / 核心信息 / 表述 / 领悟 / 其他
- 只留值得保存的条目，每条只给一个分类
- 没有条目的分类直接省略，同一分类的归到一起输出

[输出] 所有条目一律用层级式 bullet
- 上层 bullet 只放标题，说明放进下层 bullet
- 活下来的条目要能单独读懂，用大白话
- [对象的本质与原理] 以"本质：X 是一个……的器官/系统/结构"开头
- [表述] 给出标题 + 中文含义 + 原文（中文加注拼音）
```

重要的对话，分成三段各跑一次。一口气交代下去，丢弃的力度就软了。

## 值钱的是那条丢弃规则

这套系统的价值，不在花哨的输出。它在那条规则上：让机器替你判断"什么不该留"。

AI 以后只会生成得越来越多。越是这样，变得稀缺的就不是生成，而是会挑、会扔。所以我把这套丢弃标准，当代码一样做版本管理。
