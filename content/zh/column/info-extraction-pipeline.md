跟 AI 聊得越多，手里真正留下的知识反而越少。听起来像反话，可道理很简单：AI 生成的量，远超我能核对、能整理的量。好话源源不断地涌出来，可一个月后能再找回来的，几乎没有。

所以大多数"用 AI 整理笔记"的尝试都失败了。大家都把劲使在*抽得更多*上。抽得越多，笔记就越快变成垃圾场。十条备忘里有九条再也不会被翻开。

我走了相反的路。我没做一个抽得更多的工具，而是做了一个狠下心丢弃的工具。这套系统真正的内核不是它的提取能力，而是*丢弃的标准*。

## 先丢弃，再提取

我这套提取纪律里的第一条规则，永远从"扔掉"开始。

- 拿不准就扔
- 不过度提取、不做对话摘要、不留泛泛而谈
- 凡是搜一下就能轻松找回的信息，扔掉（这是最强的一道过滤）
- 凡是去掉专有名词就变成常见套话的内容，扔掉
- 没有值得保存的，就只输出"无内容可保存"
- **漏掉，也好过过度提取**

只有六样东西能留下来，而且只在它明确符合时：真正要去做的行动、新手不会知道的信息、某套系统的结构和运作逻辑、能原样复用的表述、可以当证据的事实，以及足以改变你判断标准的领悟。

## 为什么不一次做完，要分三段

让 LLM"把这段对话里重要的部分整理一下"，一次性交代下去，它会很热心地把一切都整理出来。问题就在这儿。当提取、解读、整理一次性全做完，丢弃这一步根本就不会启动。

所以我把它拆成几段，逼模型在每一段都再丢弃一次。第一段只抽候选 —— 不做整理。第二段解读候选、赋予含义，但凡是掺了推断的，都得标成"推测"。第三段把一切整理成可以直接粘进 Notion 的层级式 bullet，顺手再丢弃最后一次。能活下来的，必须脱离上下文、单独读也能看懂。

最关键的那个机关，就是逼模型标出某项到底是不是推测。它能阻止 AI 把自己编出来的结构，和对话里真有依据的结构混在一起。

## 复制即用的提示

### 第一段 —— 候选提取

```text
从下面的对话/文本里，只抽出值得保存的"候选"。
这是第一遍候选提取，不是最终成稿。别去打磨分类，也别润色句子。
关键是狠狠丢掉低价值的东西，只留下值得在第二段去解读的候选。

[最高原则]
- 拿不准就扔 / 不过度提取 / 不做对话摘要 / 不留泛泛而谈
- 搜一下就能轻松找回的信息，扔掉
- 没有的内容不要编造
- 没有值得保存的，就只输出"无内容可保存"
- 条目数量要少，意思相同的合并掉

[要留的] 仅当它明确符合下列其一时
1) 真正要去做的行动  2) 新手不会知道的重要信息
3) 某套系统的结构 / 流程 / 分支 / 运作逻辑  4) 能原样复用的表述
5) 可以当证据的事实  6) 改变判断标准的领悟

[要丢的] 感想 / 附和 / 中间过渡的解释 / 单纯的答复摘要 / 通用常识 /
不显露结构的零碎片段 / 去掉专有名词就变成套话的内容

[输出] 不要分类 —— 只把候选列出来。每条临时贴一个标签：
[行动] [隐藏信息] [结构] [表述] [证据] [领悟]
- 每条 1~3 句 / 只用 bullet / 表述类候选原文照搬，一字不改
```

### 第二段 —— 赋予含义

```text
上面是第一段的候选提取结果。现在逐条解读每个候选，给它赋予"含义"。
这还不是把东西归档码放的阶段。要解读它是什么、为什么重要、显露出什么结构。

[原则]
- 哪怕是第一段的候选，价值低的在这一步再丢一次
- 结构、运作原理只在对话直接支撑的范围内推断
- 只要掺进了推断，就标成"推测"
- 落到泛泛而谈这个层次的，扔掉

把每个候选判定为下列最终类型中的唯一一种：
Action item / 对象的本质与原理 / 核心信息 / 表述 / 证据 / 领悟 / 其他

[输出格式] 候选标题
  - 最终类型: ……
  - 核心含义: ……
  - 为什么重要: ……
  - 结构 / 运作原理: ……（适用时）
  - 是否推测: 直接有据 / 部分推测
```

### 第三段 —— 供 Notion 码放的最终整理

```text
上面是第二段赋予含义的结果。现在做成可以直接挪进 Notion 的最终整理。
低价值的条目，最后再丢一次。

[原则] 拿不准就扔 / 不过度提取 / 不留泛泛而谈 /
不要为了填满分类硬留条目 / 没有就输出"无内容可保存"

[分类] Action item / 对象的本质与原理 / 核心信息 / 表述 / 领悟 / 其他
- 只留值得保存的条目，每条只给一个分类
- 没有条目的分类直接省略，同一分类的归到一起输出

[输出] 所有条目一律用层级式 bullet
- 上层 bullet 只放标题，说明放进下层 bullet
- 活下来的条目要能单独读懂，用大白话
- [对象的本质与原理] 以"本质：X 是一个……的器官/系统/结构"开头
- [表述] 给出标题 + 中文含义 + 原文（中文加注拼音）
```

重要的对话，我建议把这三段分开各跑一次。一次性交代下去，丢弃的力度就软了。

## 写在最后

这套系统的价值，不在花哨的输出，而在那条纪律 —— 让机器替你判断*什么不该留*。AI 只会越生成越多，而它越是如此，变得稀缺的就不是生成，而是筛选和丢弃。我把这套丢弃标准当代码一样做版本管理。
